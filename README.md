# HUC Internship – Git & GitHub Summary

## What is Git?
Git is a version control system used to track changes in files and code. It helps developers save different versions of a project so that they can go back to previous work if needed. Git works locally on a computer and keeps a history of all changes.

## What is GitHub?
GitHub is an online platform that hosts Git repositories on the internet. It allows developers to store their projects, share them with others, and collaborate as a team. GitHub also provides tools for reviewing code and managing projects.

## Difference Between Git and GitHub
Git is a tool that manages version control on a local system, while GitHub is a cloud-based service that stores Git repositories. Git works offline, but GitHub requires the internet to share and collaborate.

## Why Git and GitHub are Used
Git helps developers manage changes in code safely and efficiently. GitHub helps teams collaborate, share code, track issues, and work together on the same project from different locations.

## Basic Git Workflow
First, a repository is created. Changes are made to files and saved using commits. The project can be pushed to GitHub so others can access it. If changes are made online or by others, they can be pulled back to the local system.

## How GitHub Helps in Team Collaboration
GitHub allows multiple people to work on the same project using branches. Team members can review each other’s work using pull requests and merge changes safely without affecting the main code.
